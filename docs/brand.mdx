## Brand idea

VRT is the verification checkpoint between an agent's code diff and CI. The raster brand mark combines three ideas:

- graphite diff rails entering a runtime gate
- a cobalt evidence aperture that routes the plan forward
- one amber checkpoint for residual risk

## Growth use

Lead with the sharp wedge: "Stop making coding agents wait for the wrong checks." Use the proof line when space is tight: "Skipped is not passed." Avoid claiming VRT makes tests faster. The durable claim is that VRT plans the smallest serious local verification path and preserves auditable evidence before CI.

## Do not

- Do not use neon green as the main brand color.
- Do not use purple-blue gradient wallpaper as the primary visual language.
- Do not present skipped checks as passed.
- Do not replace the mark with generic checkmarks, shields, or CI pipeline icons.
